# 跑步数据分析工具

解析 Garmin FIT 文件，生成 HTML 可视化对比报告。

## 代码架构

采用 **6 层分层架构**，自底向上为：配置 → 解析 → 统计 → 格式化 → 图表 → HTML 组装。每层职责单一，通过清晰的数据流串联。

```
┌─────────────────────────────────────────────────────┐
│  8. 入口 (main)                                     │  CLI 入口，扫描文件、调用 generate_report
├─────────────────────────────────────────────────────┤
│  6. HTML 组装层                                     │  CSS / 表格行 / 完整文档拼接
│     generate_report → _assemble_full_html           │
│     _build_core_metric_rows                         │  核心指标表格 <tr> 生成
│     _build_per_km_data_rows                         │  每公里表格 <tr> 生成（含 col-cum/row-even 类）
│     _get_css_styles                                 │  内联样式（响应式布局）
├─────────────────────────────────────────────────────┤
│  5. 图表生成层                                      │  统计数据 → Chart.js JS 代码
│     build_all_chart_scripts                         │  遍历 CHART_DEFS 生成所有图表
│     _build_one_comparison_chart                     │  单个图表：数据过滤→降采样→Y轴范围→JS
│     _render_comparison_chart_js                     │  渲染 Chart.js new Chart(...) 字符串
├─────────────────────────────────────────────────────┤
│  4. 格式化层                                        │  数值 → 展示字符串
│     format_pace                                    │  配速: 6'18"
│     format_duration                                │  时长: H:MM:SS
│     format_metric_value                            │  根据 CORE_METRIC_DEFS.fmt 格式化
│     format_metric_diff                             │  差异计算与格式化（含阈值隐藏）
│     format_km_hr_diff / format_km_pace_diff /      │  每公里专用差异格式化
│       format_km_cumtime_diff                        │  HR 用 round 后求差确保一致性
├─────────────────────────────────────────────────────┤
│  3. 统计计算层                                      │  原始记录 → 聚合指标
│     compute_run_stats                               │  主函数：字段提取 → 平均值 → 衍生指标
│       ├─ _compute_field_averages                    │  各字段均值（含 cadence×2、stride×10 缩放）
│       └─ _compute_derived_metrics                   │  配速/爬升/时长/累积用时序列
│     compute_per_km_split                            │  按公里分箱：HR/pace/cum_avg_HR/cum_avg_pace
│     get_cumulative_time_at_kms                      │  整数公里节点累积用时（含容差匹配）
│     downsample / compute_y_axis_range               │  降采样 & Y轴分位数范围
├─────────────────────────────────────────────────────┤
│  2. 数据解析层                                      │  FIT 二进制 → Python 字典列表
│     parse_fit_file                                  │  按 record/session/lap 分组返回
├─────────────────────────────────────────────────────┤
│  1. 配置层                                          │  纯数据定义，无业务逻辑
│     RECORD_FIELD_MAP                                │  FIT 字段 → 内部变量名映射
│     CORE_METRIC_DEFS                                │  左侧指标表格列定义（12 项）
│     CHART_DEFS                                      │  4 个图表配置（含 filter_fn/reverse_y）
│     DOWNSAMPLE_STEP = 10                            │  图表降采样间隔
└─────────────────────────────────────────────────────┘
```

### 设计原则

- **每个函数只做一件事**：命名即文档，如 `format_km_hr_diff` 明确表示"每公里心率差异格式化"
- **展示逻辑集中**：所有格式化、差异计算、高亮判断都在第 4 层格式化层
- **配置驱动**：指标定义、图表配置均以数据结构声明，新增指标/图表只需改配置
- **容差设计**：`get_cumulative_time_at_kms` 对终点距离略小于整数公里的情况（如 15.98km vs 16km）使用 50m 容差回退

### 关键实现细节

- **心率差异一致性**：每公里 HR diff 使用 `round(new) - round(old)` 而非原始浮点差，确保与显示的整数值一致
- **配速过滤**：速度转配速时只接受 `[5, 10] min/km]` 范围，排除 GPS 漂移/停止走动异常值
- **Y 轴分位数**：图表 Y 轴基于 5%-95% 分位数计算范围，避免极端值拉偏
- **CSS 类选择器**：累积列使用显式 `class="col-cum"` 而非 `nth-child`，避免 `rowspan=2` 表头导致的列偏移问题
- **距离截断**：`max_distance_m=16000m` 截断冷却步行阶段数据

## 功能需求

### 数据解析
- 解析 Garmin FIT 格式的跑步数据文件
- 支持多文件对比分析（取 data 目录下最新的两个 .fit 文件）
- 提取核心指标：距离、时长、配速、心率、步频、步幅、触地时间、垂直比例、功率、温度、爬升等

### 可视化报告
- 左右分栏布局：左侧核心指标表格，右侧图表区域
- 图表一行两个（2x2 网格），响应式适配不同屏幕
- 大屏幕充分利用宽度，无多余留白
- 子页面顶部包含 **返回首页** 链接，方便导航
- 日期在子页面中使用短格式显示（如 `0407` vs `0404`），节省空间

### 导航页 (index.html)
- 横向表格汇总所有跑步的核心指标
- 日期为行（最新在上），指标为列，保留完整日期格式（如 `20260407`）
- 包含开始时间（北京时间 UTC+8）、总距离、时长、配速、心率、步频等全部核心指标
- 日期可点击跳转到对应详细对比报告
- 数据需手动更新或通过脚本自动生成

### 核心指标表格（左侧）
- 按分组展示：基础数据 → 心率 → 步态 → 环境
- 三列：指标名、旧数据(date1)、新数据(date2)、**差异**（新 - 旧方向）
- 差异显示规则：
  - 零值或近零值自动隐藏
  - 时长差异显示秒数（如 `-75s`）
  - 配速差异显示秒数（如 `-9s`）
  - **精确差异**：平均心率、触地时间、垂直比例、步频、最高心率、功率、温度、步幅、爬升等指标，即使显示值相同也会展示高精度差值（如 `-0.11 bpm`），揭示舍入隐藏的微小变化
  - 其他按原精度显示
- 对比高亮更优数据
- 时长格式 `H:MM:SS`（如 `1:41:52`），配速格式 `M'SS"`（如 `6'18"`）

### 图表区域（右上，4 个图表）
- 心率变化对比
- 配速变化对比（Y 轴反向，越快越高）
- 触地时间对比
- 步频对比
- Y 轴使用 5%-95% 分位数过滤极端值
- Chart.js v3.9.1 渲染

### 每公里数据表格（底部）
- 每行一公里，包含以下列：
  | 分组 | 列 |
  |------|-----|
  | 基础 | 公里编号 |
  | 心率 | date1 / date2 / 差异(bpm) |
  | 配速 | date1 / date2 / 差异(s) |
  | **累积平均心率** | date1 / date2 / 差异(bpm) — **浅蓝色背景区分** |
  | **累积平均配速** | date1 / date2 / 差异(s) — **浅蓝色背景区分** |
  | 累积用时 | date1 / date2 / 差异(s) |
- **双数公里行整行浅灰背景**
- 差异计算与核心指标一致（新 - 旧，四舍五入后比较）

## 使用方法

```bash
# 安装依赖
pip install -r requirements.txt

# 将 FIT 文件放入 data 目录（至少两个）
# 运行分析
python analyze_jogging.py

# 生成 HTML 报告（以较新日期命名，如 20260404.html）
```

## 项目结构

```
├── analyze_jogging.py   # 主脚本（~950 行，6 层架构）
├── index.html           # 导航页（核心指标汇总表）
├── requirements.txt     # Python 依赖（fitparse）
├── data/                # FIT 文件目录（git 忽略）
│   ├── *.fit
└── *.html               # 生成的对比报告（可提交）
```

## 隐私说明

FIT 文件包含位置数据（GPS 坐标），已通过 `.gitignore` 排除 `data/` 目录。
生成的 HTML 报告不含隐私数据，可安全分享。

# 代理订阅合并工具需求文档

## 1. 项目概述
创建一个工具，用于合并两个 Clash 订阅链接，并自定义路由规则。

## 2. 输入源

### 2.1 订阅源
订阅链接配置在 `.env` 文件中（不会提交到 GitHub），包含：
- **主配置（Main Config）**: Clash 订阅链接，用于 AI 服务等一般用途
- **视频配置（Video Config）**: Clash 订阅链接，用于 YouTube 等视频流媒体

**注意**: 请勿将 `.env` 文件提交到代码仓库，因为它包含敏感信息。

### 2.2 缓存功能
支持订阅缓存功能，避免重复下载订阅配置。

**缓存配置**：
- `USE_CACHE`: 是否使用缓存（默认 `false`）
  - `true`: 使用缓存，如果缓存文件存在则不重新下载
  - `false`: 每次都重新下载

**缓存目录**：
- 缓存文件保存在 `./cache/` 目录
- 文件命名格式：`{订阅名称}.yaml`（如 `ikuuu.yaml`、`Cloudfare.yaml`）
- `cache/` 目录已添加到 `.gitignore`，不会提交到 Git

**使用场景**：
- **调试时**：设置 `USE_CACHE=true`，首次运行后缓存订阅配置，后续运行直接使用缓存
- **正常使用时**：设置 `USE_CACHE=false`（或不设置），每次都下载最新配置
- **清理缓存**：删除 `./cache/` 目录即可

### 2.3 rules.conf 自定义规则
支持通过 `rules.conf` 文件添加自定义规则，这些规则具有最高优先级。

**文件位置**: `./rules.conf`

**规则格式**: 标准 Clash 规则格式，每行一条规则：
```
IP-CIDR,140.112.161.116/32,DIRECT
DOMAIN,ocw.aca.ntu.edu,DIRECT
DOMAIN-SUFFIX,windows.com,DIRECT
```

**规则优先级**: 最高优先级，在所有规则之前匹配

**冲突处理**: 如果原配置中存在相同目标的规则（域名/IP段），会被自动过滤掉

**使用场景**:
- 需要强制某个域名或 IP 段走直连（DIRECT）
- 需要覆盖原配置中的规则
- 需要添加个人化的路由规则

**示例**:
```conf
# IPTV 相关域名直连
DOMAIN,epg.112114.eu.org,DIRECT
DOMAIN-SUFFIX,163189.xyz,DIRECT

# Windows 和 Microsoft 服务直连
DOMAIN-SUFFIX,windows.com,DIRECT
DOMAIN-SUFFIX,microsoftonline.com,DIRECT

# Apple 服务直连
DOMAIN-SUFFIX,apple.com,DIRECT
```

### 2.4 命名规范
- 主配置的节点命名为: `{主配置名称} | 节点原名称`
- 视频配置的节点命名为: `{视频配置名称} | 节点原名称`
- 具体名称在 `.env` 文件中的 `MAIN_CONFIG_NAME` 和 `VIDEO_CONFIG_NAME` 配置

### 3.3 规则优先级
规则按以下优先级排列（从高到低）：

1. **自定义规则（rules.conf）** → 自定义目标 ⭐⭐ 最高优先级
   - 从 `./rules.conf` 文件加载的自定义规则
   - 优先级最高，在所有规则之前匹配
   - 冲突处理：会自动过滤掉主配置中相同目标的规则
   - 常见用途：强制直连特定域名或 IP 段

2. **国内流量规则** → DIRECT（直连）
   - `GEOIP,CN,DIRECT,no-resolve`（国内 IP 段直连）
   - 仅依赖 IP 地理位置判断，不包含域名规则

3. **视频流量规则**（YouTube + X.com）→ Video-Group
   - 186 条规则
   - 默认走 Video-Auto（视频配置节点自动选择）

4. **AI 服务规则**（OpenAI/Gemini/Claude/Grok）→ AI-Select
   - 54 条规则
   - 默认走主配置非香港节点自动选择

5. **其他外网流量**（主配置保留的所有规则）
   - 包括主配置中的所有规则（除了被过滤的重复规则）
   - 例如：
     - bilibili.com、bilibili.tv → `🌏 爱奇艺&哔哩哔哩` 或 `🇨🇳 国内网站`
     - 其他国内网站 → `🇨🇳 国内网站` → DIRECT
     - 其他外网 → `Global-Group` → Auto-Select

6. **漏网之鱼** → 🐟 漏网之鱼 → Global-Group
   - 兜底规则，匹配所有未匹配的流量

**说明**：
- 国内流量（中国 IP 段）优先匹配，确保直连不经过代理
- 完全基于 IP 地址和地理位置判断，不依赖域名规则
- 适用于任何域名（包括使用海外 CDN 的国内站点，如果 IP 在国内则直连，在海外则走代理）

**冲突解决策略**：
当下载的原配置中包含规则时，系统会自动处理冲突：
1. **自动过滤与 rules.conf 冲突的规则**：
   - 检测规则目标（域名/IP段）是否与 rules.conf 中的目标相同
   - 如果相同，则自动过滤掉原配置中的该规则
   - 确保 rules.conf 的自定义规则具有最高优先级
2. **自动过滤原配置中的国内规则**：
   - `GEOIP,CN` 规则（国内 IP 段）
   - 私有 IP 段（`10.x.x.x`, `172.16.x.x`, `192.168.x.x`, `127.x.x.x`）
3. **自动过滤原配置中的重复规则**：
   - YouTube 相关规则（`youtube` 关键词）
   - AI 服务相关规则（`openai`, `chatgpt`, `claude`, `anthropic`, `gemini`, `bard`）
4. **自动过滤原配置中的 MATCH 规则**：
   - 避免与自定义的 `MATCH,🐟 漏网之鱼` 规则冲突
5. **保留原配置中的所有其他规则**：
   - 保留 `DOMAIN`、`DOMAIN-SUFFIX`、`DOMAIN-KEYWORD` 规则（如 bilibili.com 等）
   - 保留 `IP-CIDR` 规则（IP 段规则）
   - 保留其他 `GEOIP` 规则（非 CN）
6. **修复代理组引用**：
   - 将不存在的代理组替换为 Global-Group
   - 将 `🔰 选择节点` 替换为 Auto-Select
   - 为主配置保留的代理组中的节点名称添加前缀（如 `🇭🇰 香港W01` → `ikuuu | 🇭🇰 香港W01`）
   - 移除主配置中的 `🔰 选择节点` 代理组

这样可以确保规则清晰简洁，保留主配置的域名规则（如 bilibili.com），避免 YouTube 和 AI 相关规则冲突。

## 4. 输出要求

### 4.1 合并后的配置
- 输出格式: Clash 配置文件（YAML）
- 输出路径: `/proxy/merged.yaml`

### 4.2 配置结构
- **proxies**: 合并两个订阅的所有节点
  - 主配置节点
  - 视频配置节点

- **proxy-groups**:
  - **自定义代理组**（7 个）：
    - `Auto-Select`: 包含所有节点（主配置 + 视频配置），自动选择最快节点，作为全局默认
    - `Video-Auto`: 仅包含视频配置节点，自动选择最快节点，专门用于视频流量
    - `Video-Group`: 手动选择组，默认走 Video-Auto，可切换到 DIRECT 或指定节点
    - `AI-Select`: 仅包含主配置非香港节点，自动选择最快节点，专门用于 AI 服务
    - `AI-NonHK-Group`: 手动选择组，包含主配置中非香港节点，可手动选择指定节点
    - `Global-Group`: 手动选择组，默认走 Auto-Select，可切换到 DIRECT 或指定节点
    - `🐟 漏网之鱼`: 兼容组，指向 Global-Group（保持与原配置兼容）
  - **主配置保留的代理组**：保留主配置中的所有代理组（除了与自定义代理组重名的，以及 `🔰 选择节点`），如：
    - `🌏 爱奇艺&哔哩哔哩`: 哔哩哔哩、爱奇艺等国内视频网站
    - `🇨🇳 国内网站`: 其他国内网站
    - 其他主配置中的代理组（`🔰 选择节点` 已被移除，所有引用改为 Auto-Select）

- **rules**:
  - 自定义规则（rules.conf）→ 自定义目标（最高优先级）
  - 国内直连规则 → DIRECT
  - YouTube 规则 -> Video-Group（默认走 Video-Auto，即视频配置自动选择）
  - AI 服务规则 -> AI-Select（主配置非香港节点自动选择）
  - 主配置原有规则 -> Global-Group（默认走 Auto-Select，即所有节点自动选择）
  - 保留原有规则中的所有规则，修复引用不存在的代理组问题
  - 将 `🔰 选择节点` 替换为 Auto-Select

## 5. 功能需求

### 5.1 下载订阅
- 支持从两个 URL 下载 Clash 订阅配置
- 处理网络请求异常
- 支持超时设置

### 5.2 配置解析
- 解析 YAML 格式的 Clash 配置
- 提取 proxies、proxy-groups、rules 信息

### 5.3 配置合并
- 合并 proxies 列表
- 合并 proxy-groups（如需要）
- 合并并优化 rules

### 5.4 规则处理
- 加载自定义规则文件（rules.conf）
- 插入 YouTube 优先规则
- 插入 AI 服务规则
- 去除主配置中重复的 YouTube 和 AI 相关规则
- 冲突检测：自动过滤掉主配置中与 rules.conf 相同目标的规则
- 确保规则顺序正确（优先级从高到低）

### 5.5 节点处理
- 从主配置节点中识别香港节点（节点名包含 "HK"、"Hong Kong"、"香港" 等关键词）
- 将非香港节点归类到 AI-NonHK-Group
- 为 AI-Select 组启用自动选择功能（url-test 或 similar）

### 5.6 配置输出
- 生成标准的 Clash YAML 配置
- 保存到指定路径
- 支持配置验证

## 8. 其他说明

### 8.1 默认路由策略
- **YouTube 流量**: 走视频配置节点（速度快，适合视频流媒体）
- **AI 服务流量**: 走主配置非香港节点（稳定性好，避免特定 IP 被限制）
- **其他外网流量**: 走所有节点中的最快节点（全局自动优化）
- **国内流量**: 直连（不经过代理）

### 8.2 代理组说明（大白话版）

**1. Auto-Select（自动选择所有节点）**
- 这是"自动选择全部"组
- 里面包含了所有的节点（主配置 + 视频配置）
- 每5分钟自动测试一次哪个节点最快，就自动切换到那个节点
- 这是其他大部分网站的默认选择

**2. Video-Auto（视频专用自动选择）**
- 这是"视频自动选择"组
- 里面只包含视频配置的节点
- 每5分钟自动测试视频配置哪个节点最快
- 专门给视频流量（YouTube、X.com 等）使用，因为视频配置节点看视频速度好

**3. Video-Group（视频手动选择）**
- 这是"视频手动切换"组
- 默认选项是 Video-Auto（自动选择最快的视频配置节点）
- 你也可以手动改成 DIRECT（直连，不翻墙）或者选择具体的某个视频配置节点
- 如果你不想自动切换，就手动选一个固定的节点

**4. AI-NonHK-Group（AI 手动选择）**
- 这是"AI 手动切换"组
- 里面只包含主配置的非香港节点（日本、美国、新加坡等）
- 你可以手动选择具体用哪个节点访问 AI 服务
- 默认不使用，AI 服务会走 AI-Select

**5. AI-Select（AI 专用自动选择）**
- 这是"AI 自动选择"组
- 里面只包含主配置的非香港节点
- 每5分钟自动测试哪个节点最快
- 专门给 AI 服务用（ChatGPT、Claude、Gemini 等），因为用特定地区的 IP 不容易被限制

**6. Global-Group（全局手动选择）**
- 这是"全局手动切换"组
- 默认选项是 Auto-Select（自动选择最快的所有节点）
- 你也可以手动改成 DIRECT（直连）或者选择具体的某个节点
- 如果你想全局固定用一个节点，就在这里选

**7. 🐟 漏网之鱼（兼容组）**
- 这是一个"兼容"组
- 直接指向 Global-Group
- 主要是为了保持和原配置兼容，实际用不到

### 8.3 使用场景举例

**场景1：正常使用**
- 所有外网都走 Auto-Select 自动选择，无需任何操作
- YouTube 自动走视频配置最快节点
- AI 服务自动走主配置非香港最快节点

**场景2：发现某个节点特别快，想固定用**
- 打开 Clash 客户端
- 找到 Global-Group，把选项从 Auto-Select 改成你想要的那个节点
- 这样所有外网都会固定走这个节点

**场景3：YouTube 卡顿**
- 打开 Clash 客户端
- 找到 Video-Group
- 如果现在是 Video-Auto，说明自动选择的节点有问题
- 你可以手动选择一个固定的视频配置节点

**场景4：AI 服务访问慢**
- 打开 Clash 客户端
- 找到 AI-NonHK-Group
- 手动选择一个日本或美国节点
- 或者等5分钟，AI-Select 会自动切换到更快节点

## 9. 安全性说明

### 9.1 敏感信息保护
本项目使用环境变量来存储敏感信息（订阅链接），确保不会泄露到代码仓库。

### 9.2 文件结构
```
proxy/
├── .env                  # 环境变量文件（包含敏感信息，不提交到 Git）
├── .env.example          # 环境变量模板（提交到 Git，作为示例）
├── .gitignore            # Git 忽略规则（确保敏感文件不被提交）
├── cache/                # 订阅缓存目录（不提交到 Git）
│   ├── ikuuu.yaml       # ikuuu 订阅缓存
│   └── Cloudfare.yaml  # Cloudfare 订阅缓存
├── merge.py              # 主程序（不包含敏感信息）
├── requirements.txt      # Python 依赖
├── rules.conf            # 自定义规则文件（最高优先级规则）
├── merged.yaml           # 生成的配置文件（不提交到 Git）
└── 需求文档.md            # 项目文档
```

### 9.3 安全措施
1. **环境变量隔离**: 所有订阅链接存储在 `.env` 文件中
2. **.gitignore 保护**: `.env` 和 `merged.yaml` 已添加到 `.gitignore`
3. **模板文件**: 提供 `.env.example` 作为配置示例，不包含真实信息
4. **代码检查**: 代码中不包含任何硬编码的敏感信息

### 9.4 使用指南
1. 首次使用时，复制 `.env.example` 为 `.env`:
   ```bash
   cp .env.example .env
   ```

2. 编辑 `.env` 文件，填入真实的订阅链接

3. 安装依赖:
   ```bash
   pip install -r requirements.txt
   ```

4. 运行脚本:
   ```bash
   python3 merge.py
   ```

5. （可选）使用缓存功能：
   - 在 `.env` 文件中设置 `USE_CACHE=true`
   - 首次运行会下载并缓存订阅配置
   - 后续运行会直接使用缓存，不重复下载
   - 清理缓存：删除 `./cache/` 目录
